## Severity grid

- **P0** — shipping bug, crash risk, or user-visible data desync.
- **P1** — functional gap, broken handler, misleading copy, or integration gap.
- **P2** — polish: a11y, dead code, performance hygiene, minor DX.

## P0 — Ship-blocking correctness

### P0-1 · TS compilation errors in test files

`pnpm exec tsc --noEmit` reports 4 errors — two test helpers spread
`...overrides` **after** explicit defaults, causing TS2783.

- `src/lib/engine/pairing-rationale.test.ts` L17, L18, L21
- `src/lib/engine/preference-observations.test.ts` L18

### P0-2 · Favorites page empty state lies

`/path/favorites` lists `useCookSessions.favoriteSessions` but the empty
copy tells users "tap the heart on any dish" — the Today heart writes
to a different store (`useSavedDishes`). Users follow instructions, see
nothing, assume app is broken.

- `src/app/(path)/path/favorites/page.tsx` L84-86 (copy)
- `src/components/today/quest-card.tsx` (uses `useSavedDishes`, the OTHER store)

### P0-3 · Reroll can hang forever on `success: false`

`ResultStack.useEffect` clears `rerollingIndex` on `isError` or on
`success && side`, but NOT on `{ success: false, side: null }` returned
by `pairing.rerollSide` when no alternatives exist. User is stuck
staring at a spinner.

- `src/components/today/result-stack.tsx` L117-151
- `src/lib/trpc/routers/pairing.ts` L238-244

### P0-4 · Per-dish mistake suppression silently disabled

Sprint 20C built `useMistakeSuppression` with per-dish scoping. Neither
cook page passes `dishSlug` into `StepCard`, so `MistakeChip`'s dismiss
button never renders in production. Three sprints of work behind a
missing prop.

- `src/app/cook/[slug]/page.tsx` L481-505 (single cook — no dishSlug)
- `src/app/cook/combined/page.tsx` L683-711 (combined — no dishSlug)
- `src/components/guided-cook/mistake-chip.tsx` L36-38, L77-93

### P0-5 · Combined cook wins cannot capture low-star feedback

Sprint 20D P14 wired `onFeedback` chips; only single-cook page passes
the callback. Combined sessions silently discard low-star feedback.

- `src/app/cook/combined/page.tsx` L732-759 (no `onFeedback`)
- `src/app/cook/[slug]/page.tsx` L521-522 (has it)

## P1 — Functional gaps + integration

### P1-1 · Combined Grab missing `CookWatchlist`

Single cook renders the mistake watchlist above ingredients; combined
cook doesn't.

- `src/app/cook/combined/page.tsx` L584-595
- `src/app/cook/[slug]/page.tsx` L432-436 (reference)

### P1-2 · Combined Grab loses substitution memory

`IngredientList` in combined mode receives no `dishSlug`, so the 90-day
swap memory we just shipped (P13) is a no-op there.

- `src/app/cook/combined/page.tsx` L586-594

### P1-3 · `DishTransitionCard` "dish N of N" off-by-one

After finishing dish 1, `currentDishIndex` is still 0. Card reads "dish
1 of N" under the _next_ dish's title — classic off-by-one.

- `src/app/cook/combined/page.tsx` L715-727

### P1-4 · Coach quiz state desync

`CoachQuiz.onClose` writes `sous-coach-quiz-done` to localStorage but
never calls `setQuizDone(true)`. The deferred effect that reads it has
already fired, so "Personalize" stays visible in More options for the
rest of the session.

- `src/app/(today)/today/page.tsx` L99-119, L647-658

### P1-5 · Today loading + error stack when pairing fails

Transition to `"results"` only runs on successful data. On error, `view`
stays `"loading"`, so the skeleton and error blocks render together.

- `src/app/(today)/today/page.tsx` L191-203, L507-528, L622-641

### P1-6 · Speech synthesis bleeds across steps

`StepCard.handleReadAloud` has no cleanup. Advancing steps while TTS
is playing reads the old step over the new screen.

- `src/components/guided-cook/step-card.tsx` L108-126

### P1-7 · `/sides` navigates during render

Empty `main` query calls `router.replace("/today")` inside render body.
React yells in dev; can cause double nav.

- `src/app/sides/page.tsx` L133-136

### P1-8 · Flavor Pairs score mismatch

Win screen displays `1000 - moves*50 - elapsed*5`; `recordScore` stores
`1000 - (moves+1)*50 - elapsed*5` via stale closure. Leaderboard and
victory screen disagree.

- `src/app/games/flavor-pairs/page.tsx` L127-131, L207-208

### P1-9 · `fuzzySearch.getSuggestions` is non-deterministic

`.sort(() => Math.random() - 0.5)` — statistically biased AND breaks
test determinism.

- `src/lib/fuzzySearch.ts` L58-60

### P1-10 · Combined session `recipeSlug` is only first dish

Session row for a combined cook records only the main dish slug. History
queries by slug miss multi-dish context.

- `src/app/cook/combined/page.tsx` L184-188

## Fix plan — execution order

**Batch A (P0 hot-fixes, all must land):**

1. Fix the 4 TS compile errors (rewrite test helpers without trailing spread).
2. Rewrite Favorites empty-state copy to match reality (or actually unify the two stores; we'll pick copy fix for minimal surface-area).
3. `ResultStack` reroll: clear spinner on `success: false` too.
4. Thread `dishSlug` into `StepCard` (both single + combined cook pages).
5. Wire `onFeedback` into combined-cook `WinScreen`.

**Batch B (P1 integration + bugs):** 6. Combined Grab: add `CookWatchlist` + pass `dishSlug` to `IngredientList`. 7. Fix `DishTransitionCard` off-by-one (show `index + 2` for "up next" — or restructure to show dish being introduced). 8. Coach quiz: `onClose` updates `quizDone` in component state. 9. Today: prevent loading + error overlap by forcing view→error on `pairingQuery.isError`. 10. Cancel speech synthesis on `stepNumber` change + unmount. 11. `/sides` empty-main redirect moved into `useEffect`. 12. Flavor Pairs: unify score formula + use ref for recording. 13. `fuzzySearch.getSuggestions`: deterministic slice, not random sort. 14. Combined session `recipeSlug` encode as `main:side1:side2` or add `combinedSlugs[]` field. 15. `CookForTwoChip`: replace inner role=button with a sibling button. 16. `ResultRow`: drop the fabricated "~15 min" fallback copy. 17. `CameraInput`: replace `alert()` with inline-error state. 18. `SkillDetailSheet`: deterministic pick (first dish) for "Practice again."

**Batch C (P1 dead code removal, large):** 19. Delete: `bread-quiz.tsx`, `user-avatar.tsx`, `quick-actions.tsx`, `use-recipe-overlays.ts`, `data/bread-quiz.ts`, `withFallback` export. 20. Delete dead tRPC routers + procedures: `journey`, `coach`, `content`, `pairing.explain`, `cook.start`, `cook.complete`.

**Batch D (P2 polish):** 21. A11y sweep on TabBar icons, TextPrompt label, BirdMascot, CookTimer pulse, ScrapbookEntryCard alt, AppPreviewSection alts. 22. Dedup lucide imports; remove unused `_rerollSeed`, `isRerolling`, unused `personalNote` props on StepCard. 23. StepCard progress bar clamp; treat `timerSeconds === 0` as absent timer; thread `direction` in combined. 24. `cook.complete` → return `success: false` on caught DB error (pre-delete cleanup; moot if Batch C deletes it).

**Verification gate:** lint clean, `pnpm exec tsc --noEmit` clean, all
vitest tests green, `pnpm build` clean, commit + push to main.

**Out of scope (intentional non-goals for this pass):**

- Unifying `useSavedDishes` and `favoriteSessions` into a single store — meaningful refactor; copy fix is the bounded move.
- Reconciling `useSkillProgress.totalXP` vs `useXPSystem.level` into a single XP source — documented as P0-1 by the audit but the right fix is a product decision, not a mechanical one. Leaving as a tracked item.
- Swipe-based step nav in Guided Cook (feature add, not a bug).
- Cross-tab localStorage `storage` event listeners on all hooks (broad ergonomic change).
- Cook-session resume across tab reloads (real feature, not a fix).

These non-goals are captured here so the next sprint has a clean
starting list.
